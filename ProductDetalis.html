LOGIN & ACCESS PROCESS (SHVYA INVENTORY)

Step-by-step authentication and role checks to secure access to inventory, invoices and POS modules.

1. User Opens Login Page

Landing page or app login screen (email/username input visible, “Forgot?”, 3rd-party login buttons).

2. Credential Submission

Client securely submits credentials (POST to /auth). Use HTTPS, input validation and rate-limits.

3. Authenticate Credentials

Server checks hashed password or token. If using OAuth/OIDC, validate third-party tokens.

4. Multi-Factor / Risk Checks

Trigger 2FA (TOTP/SMS/Email) or device/risk checks. Block or require verification when suspicious.

5. Role & Permission Check

Map user to roles (admin, sales, user). Evaluate permissions for modules: Inventory, POS, Invoices, Reports.

6. Session / Token Creation

Create session cookie or sign JWT (short expiry). Secure flags, HttpOnly, SameSite; refresh token strategy.

7. Access Granted

User redirected to dashboard / module (Inventory / POS / Sales) with UI tailored to permissions.

8. Logging & Monitoring

Record login events, IP, device, and auth failures. Monitor suspicious activity and alert as needed.

9. Logout / Session Revoke

Clear session and revoke refresh tokens. Optionally force global logout on credential changes.
